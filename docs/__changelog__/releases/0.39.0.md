### Added

- Settings page (v2): tabbed layout with five sections — Views, Project,
  Schedule, Tags, Links. All project.md fields are now editable from the UI.
- Settings Views tab: enabled/disabled collapsible groups, search filter,
  check all/uncheck all bulk actions.
- Settings dirty form tracking: snapshot-based comparison auto-detects when
  values revert to their original state. Dirty tabs show a warning dot and
  `beforeunload` prevents accidental navigation.
- OpenAPI (Scalar): vendored API reference at `/api/v1/reference` with
  documented schemas for all Settings, Milestone, and Portfolio endpoints.
- OpenAPI model names: all Zod schemas now register `.openapi("Name")` so
  they appear in the Swagger Models section (Task, FeaturesList,
  UpdateProjectConfig, ProjectLink, etc.).
- MCP server (v2): scaffold at `/mcp` with 7 milestone tools as thin
  wrappers over shared services. Service-first architecture with unified
  Zod schemas.
- Sidebar: dynamic navigation built from enabled features list. Toggling
  features in Settings immediately refreshes the sidebar.

### Changed

- `UpdateProjectConfigSchema` now accepts all project.md fields (startDate,
  workingDaysPerWeek, workingDays, tags, links) instead of only name,
  description, and features.
- `ProjectService.updateConfig` applies all schema fields.
- `enabledFeatures` threaded through all view routes and layout components.
