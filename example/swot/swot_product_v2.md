## Strengths

- Markdown-first architecture — no vendor lock-in
- Local-first with optional cloud sync
- MCP integration for AI-native workflows
- Full offline support
- Open-source transparency builds trust
- Sub-second page loads with SSR + htmx

## Weaknesses

- No real-time collaboration yet
- Limited third-party integrations
- Single-developer bottleneck on core features
- No mobile-native app
- Documentation gaps for advanced features

## Opportunities

- AI planning assistants becoming mainstream
- Developers moving away from bloated SaaS tools
- Growing demand for privacy-respecting tools
- Potential Obsidian/Notion plugin ecosystem
- Enterprise self-hosted deployment market

## Threats

- Notion and Linear adding AI features rapidly
- Open-source competitors (Plane, Focalboard) gaining traction
- Developer fatigue with new productivity tools
- Funding constraints limiting marketing reach
- Cloud-native expectations conflicting with local-first approach
