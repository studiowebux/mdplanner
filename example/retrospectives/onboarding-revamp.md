# Onboarding Revamp Retrospective

Review of the 3-week onboarding UX overhaul project.

## Keep Doing

- Weekly user testing sessions — caught friction points early
- Small focused PRs made code review fast and low-risk
- Design and engineering pairing on the first day of each feature
- Using feature flags to ship incrementally without blocking the release

## Stop

- Designing in isolation without dev input — caused two rework cycles
- Skipping accessibility checks until the end of a feature
- Making scope decisions in Slack threads — decisions got lost

## Try

- Invite a customer to the mid-project review for live feedback
- Run accessibility audit at component creation, not at PR review
- Use a decision log (ADR) for any scope change affecting more than one team
- Timebox design explorations to 2 days max before moving to implementation
