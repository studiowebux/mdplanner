# Open Source Strategy

## What to open source
- Core library (domain factory, repositories)
- Example data / templates
- Deployment configs (Docker Compose)

## What stays private
- Brain protocol + Claude Code integration
- Paid feature code
- Customer data tooling

## Why open source the core
- Trust signal for self-hosters
- Community bug reports = free QA
- Portfolio piece

## License
MIT for library code. Business Source License for premium features (3-year delay to MIT).
